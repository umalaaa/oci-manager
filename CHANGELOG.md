# 变更记录

## 0.3.1 (2026-02-12)

### 修复
- 登录方式校验支持读取 Profile 默认 SSH 公钥与 Root 登录配置，避免误报“至少选择一种登录方式”。

## 0.3.0 (2026-02-11)

### 新增
- Telegram 集成：按钮菜单、Profile/分区/可用区/机型/任务/实例管理，支持绑定与三次失败拉黑。
- 可用区多选并行创建：选择多个 AD 时会生成多个任务并发执行。
- 机型增强：Flex 机型支持选择 OCPU/内存，自动分页展示；机型选择后可直接进入硬盘大小选择。
- 新增硬盘大小选择（启动卷大小），可设置常用容量或恢复默认自动。
- Root 登录选项：自动生成随机 Root 密码、注入 cloud-init，并在成功通知中发送，同时写入本地日志。
- Web UI：新增 Root 登录与“使用 SSH 公钥”开关；新增创建前校验。
- 自动选择可用子网：未指定 subnet 时自动选择可用子网。

### 变更
- 强制至少一种登录方式（SSH 公钥或 Root 登录），避免创建后无法登录。
- 分区/可用区切换会清空机型缓存，避免机型与 AD 不匹配。

### 安全
- 忽略 `data/` 目录，防止绑定信息等运行态数据被误提交。
